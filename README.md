# PixelProof — Real-time Image Fact-Checking (Hackathon Edition)

PixelProof is a Chrome Manifest V3 extension prototype built for hackathons. It scans images on a page, uses an image-authenticity service and LLMs to generate human-friendly explanations, and looks for matching fact-checks.

**Short pitch:** show that you can detect likely AI-generated or manipulated images, explain the artifacts to non-technical users, and surface known fact-checks — all within a browser extension demo.

**Overview**
- Scans images in-page (single image or `Scan All`) and returns:
	- a short verdict (Likely AI / Possibly manipulated / Looks authentic),
	- a confidence score, and
	- a human-readable explanation generated by a small LLM prompt.
- Enrichment: queries Google Fact Check Tools for matching claims and shows sources.

**Quick start (for your hackathon laptop)**
- Prerequisites: Chrome (or Chromium), Node.js (optional, used for `generate_config.js`).
- Steps:
	1. Clone the repo: `git clone https://github.com/[username]/pixelproof.git`
	2. Open this folder in VS Code or terminal.
	3. Copy `.env.example` → `.env` and paste your API keys there.
		 - Windows PowerShell: `copy .env.example .env`
	4. Generate runtime config: `node scripts/generate_config.js` (writes `config.js` locally).
	5. Load the extension: `chrome://extensions` → Developer mode → Load unpacked → select `pixelproof-1` folder.
	6. Open a page with images and use the popup/context menu to scan.

**API Keys (what you need and where to get them)**
- REALITY_DEFENDER_API_KEY — Reality Defender image-auth API (https://realitydefender.com). Sign up and create an API key.
- GEMINI_API_KEY — Google Generative (Gemini) or Google Cloud API key that can be used with the Generative API/AI Studio. Create via Google Cloud Console / AI Studio.
- FACT_CHECK_API_KEY — Google Fact Check Tools API key; enable the API in Google Cloud, then create credentials.

See the example config for expected variables: [config.example.js](config.example.js#L1-L9).

**Architecture & key files**
- `background.js` — service worker orchestrator: receives scan requests, dedupes concurrent scans, calls `utils/api.js`, caches results, and replies to content/popup.
- `content.js` — injected script that locates images, attempts to convert them to base64 (canvas), and draws overlay UI per image.
- `popup/` — popup UI that can trigger `Scan All` and show a summary dashboard.
- `utils/api.js` — wrappers around Reality Defender, Gemini, and Fact Check APIs; contains simulated fallbacks and connectivity diagnostics.
- `utils/cache.js` — caches results keyed by image URL to prevent repeated calls.
- `scripts/generate_config.js` — helper that reads `.env` and writes `config.js` locally so keys are not committed. (See [scripts/generate_config.js](scripts/generate_config.js#L1-L200)).

**How scanning flows (technical)**
1. `content.js` calls `getImageAsBase64()` (best-effort; CORS/canvas tainting may block some remote images).
2. Sends `{ imageUrl, base64 }` to `background.js` via `chrome.runtime.sendMessage`.
3. `background.js` checks `utils/cache.js` for recent results; if none, calls `scanWithRealityDefender(base64)`.
4. On RD verdict → `explainWithGemini()` + `checkFacts()` enrich the verdict; results are cached and returned to the tab.

**Testing & troubleshooting**
- Manual test scenarios:
	- Single image scan (right-click or overlay button).
	- `Scan All` from popup on a page with many images.
	- With keys missing: verify simulated fallback appears (useful for offline demos).

- If you see `TypeError: Failed to fetch` when calling Reality Defender:
	1. Open the background service worker console: `chrome://extensions` → locate PixelProof → Inspect Service Worker.
	2. Reproduce the scan and copy the SW console output. Look for the `Connectivity checks` object (the extension logs the results of `https://www.google.com/generate_204` and `https://api.realitydefender.com/`).
	3. Run local connectivity checks (PowerShell):
		 - `Invoke-WebRequest -Uri 'https://www.google.com/generate_204' -UseBasicParsing`
		 - `Invoke-WebRequest -Uri 'https://api.realitydefender.com/' -UseBasicParsing`
	4. If those fail, it's a network/firewall/proxy issue — rotate networks or use a different laptop for the demo.

**Demo checklist (concise)**
- Show extension loaded in Chrome.
- Scan one image and explain verdict + Gemini explanation.
- Scan the same image again to show cache hit (fast return).
- `Scan All` on a news article; show multiple overlays and the popup summary.
- Remove `REALITY_DEFENDER_API_KEY` and show simulated fallback + diagnostics printed to SW console.
